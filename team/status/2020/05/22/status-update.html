22nd May 2020: IAC Team Status Update

May 22, 2020 • pmcmaw

The IAC team has been working on a lot of new things over the past week. We want to share with our community some pretty interesting things. Here’s a brief overview of what we’ve been up to:

Community Management Tool

We wanted to have all the information about our supported modules in one place. We decided to change the data source of the Community Tool from a local file called modules.json to the online available https://puppetlabs.github.io/iac/modules.json which is generated from the same information that is powering the team’s modules page. You can check the new updates here.

The report for our tooling has been using the same mechanism since March. We will continue to build all our future automation tooling based on this data set, so that we can update-once-run-everywhere.

PDKSync Runs

David Swan has performed 2 PDKSync runs this week to include generic template updates in our modules and to add Ubuntu 20.04 support. Danny has been running one to switch our default provisioner from VMPooler to ABS for our weekly and pre-release CI runs. Travis was busy! :D

Making it Possible to Visualize Data

DavidS has been working hard spent a few more minutes on making use of honeycomb. This week’s new query will track our migration from VMPooler to ABS. It is super exciting that we can now see dashboards and drill down rather than scrolling through large log files.

Litmus Image Creation

Since the end of 2019 we’re using custom-built images to test modules with litmus. Litmus image creation is now part of the IAC team OS certification process. As part of our work on validating all modules for Ubuntu 20.04, David Swan put in a quick PR to add it to our list of images. Being able to easily create a new Litmus image enables the IAC to team run OS certification test across multiple modules quickly. Great Job Florin on the Litmus image creation scripting.
